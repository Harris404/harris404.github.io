## 当前架构的三个核心问题

### 问题 1: iOS App 缺少真正的本地 AI 推理 ❌

**现状**：
- App 使用 Ollama API 通过网络调用 Qwen 模型
- 没有将 Qwen2.5-1.5B MLX 模型打包到 App 内
- 无法离线使用 AI 功能

**影响**：
- 必须有网络连接才能对话
- 无法满足"本地推理"的隐私承诺
- 没有发挥 MLX + Apple Silicon 的性能优势

**解决方案**：
- 实现真正的 MLX Swift 集成（mlx-swift-examples 参考）
- 打包 Qwen2.5-1.5B-Q8 模型（1.7GB）到 App Bundle
- 实现本地 Function Calling 解析

### 问题 2: RAG 向量搜索未实现 ⚠️

**现状**：
- RAG 数据库有 27,008 个 embedding
- 但 SQLite-VSS 扩展未安装
- 只能使用关键词搜索（低质量）

**影响**：
- 查询"如何申请 Medicare"只能匹配关键字，不能理解语义
- 检索准确率低于 70%（目标 90%+）
- 无法处理中英文混合查询

**解决方案**：
- 安装 SQLite-VSS 或使用 sqlite-vec
- 创建 HNSW 向量索引
- 实现相似度搜索 API

### 问题 3: 针对目标用户的内容严重不足 📉

**现状分析**：
根据 explore agent 的详细分析，你的 RAG 内容主要覆盖：
- ✅ 政府服务（签证、Medicare、Centrelink、税务）
- ✅ 医疗设施（25,848 条全国数据）
- ✅ 租房法律（4个州）

但严重缺少：

**华人/留学生/游客专属内容**：
- ❌ 中文社区资源（华人协会、中文媒体、文化活动）
- ❌ 留学生实用指南（OSHC、学生折扣、租房攻略、打工限制）
- ❌ 旅游景点和攻略（悉尼歌剧院、大堡礁、国家公园）
- ❌ 中文服务提供者（中文医生、律师、翻译、会计）

**影响**：
- 无法回答"悉尼有哪些中文教会？"
- 无法推荐"墨尔本留学生租房避坑指南"
